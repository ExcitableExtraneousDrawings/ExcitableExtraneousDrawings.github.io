Week 1

Programmeren

Het hoofdonderwerp van deze periode is programmeren. Programmeren is het proces van ontwerpen, schrijven (coden), testen en analyseren van een computerprogramma. Een computerprogramma bestaat uit een reeks van instructies in een bepaalde programmeertaal die de computer stap voor stap kan uitvoeren. De programmeur – jij, straks! – bepaalt wat die instructies zijn.

Het uitvoeren van een programma in een computer wordt gedaan door de processor (of central processing unit, CPU). Deze bestaat uit miljarden kleine schakelaars die elkaar aan (1) of uit (0) kunnen zetten: transistoren. Er loopt elektriciteit door de processor heen, en de staat (aan of uit) van de schakelaars bepaalt welke informatie er opgeslagen is en welke berekeningen er uitgevoerd worden.

In het filmpje hieronder zie je een uitvergrootte processor van 42.370 transistoren, elk met een LED-lampje, zodat het goed te zien is wat er gebeurt. Hier is ook nog een mooi visueel voorbeeld (druk op de zwarte play-knop).

De processor 'begrijpt' alleen instructies in machinetaal. Dit zijn hele simpele instructies zoals "tel deze getallen bij elkaar op" en "voer een ander deel van het programma uit". Niet erg handig om mee te programmeren. Daarom zijn er programmeertalen gemaakt zodat je de details van machinetaal niet hoeft te kennen. De programmeertaal wordt vooraf vertaald naar machinetaal door een compiler, of wordt tijdens het uitvoeren pas vertaald door een interpreter (zoals bij Python).

Er zijn veel verschillende programmeertalen. Met sommige lagere programmeertalen zoals C of C++ moet je nog veel snappen van hoe een computer werkt. Bij hogere programmeertalen zoals Python of Javascript hoef je minder van deze details te weten, en wordt het je makkelijker gemaakt. Deze talen werken op een hoger abstractieniveau. Het nadeel is wel dat deze talen soms langzamer werken of minder mogelijkheden bieden.

Waarom zou je willen leren programmeren? Mijn drie belangrijkste redenen:

  1. Het leert je op een creatieve manier problemen oplossen. Je zal problemen moeten opdelen in kleine stukjes, patronen in het probleem herkennen en instructies maken die de computer kan 'begrijpen' en uitvoeren. Er zijn veel verschillende oplossingen waarbij je creatief moet omgaan met de mogelijkheden.
  2. Kennis van programmeren kan je helpen bij elke studie of interesse. Computers zijn de werkpaarden van de 21e eeuw. Of je nou een taalkundige analyse moet maken, real-time de beurskoerzen wil bijhouden, wilt voorspellen hoe het klimaat zich zal ontwikkelen, je automatisch een spel wilt oplossen, of visuele kunst wil maken: het begint met de kennis van een programmeertaal.
  3. Er is veel behoefte aan mensen met kennis van programmeren in de arbeidsmarkt. Er valt een hoop geld te verdienen met goede kennis van programmeren. Dat hoeft niet perse als softwareontwikkelaar; ook in het (project)management, de uitvoering en ondersteuning is het goed om kennis van programmeren te hebben.

Python

Python is een programmeertaal die goed geschikt is om als eerste te leren. Ik maak graag gebruik van een cursus van de University of Waterloo. Deze legt alles helder en in het Nederlands uit, en biedt interactieve opgaven. Er is ook een handige visualisatietool, waarmee je stap voor stap door een programma kan lopen.

Programmeren is soms best lastig! Wees niet bang om een halve oplossing gewoon maar uit te proberen, tegen vage foutmeldingen aan te lopen of even vast te zitten. Tegen iemand praten over een probleem kan al helpen. Niemand aanwezig (thuis)? Zelfs praten tegen een badeendje kan helpen!

Opdracht 1: Ga naar de website van de cursus en maak de hoofdstukken "0: Hallo", "01: Variabelen" en "01E: Errors". Je mag een account maken zodat je voortgang wordt opgeslagen, maar dat hoeft niet.

GitHub

GitHub is een website die gemaakt is om makkelijk broncode en andere bestanden met elkaar te delen, en samen te werken aan een project. Het is al ruim 10 jaar de meest populaire website voor softwareontwikkelaars en artiesten om hun projecten inzichtelijk te maken voor anderen.

GitHub maakt gebruik van het programma git, een versiebeheersysteem (VCS). Met git kan je bijvoorbeeld de code van iemand anders ophalen (door een pull te doen), je wijzigingen opslaan (in een commit), en je commits versturen (met een push).

Opdracht 2: Voor latere opdrachten heb je een account op GitHub nodig. We gaan ook een aantal websites gebruiken die gekoppeld kunnen worden aan GitHub. Zo hoef je maar één keer je wachtwoord in te typen. Maak een account op GitHub aan. Kies een redelijke gebruikersnaam, vul je emailadres goed in en vergeet je wachtwoord niet.

Opdracht 3: Je krijgt een mailtje van GitHub. Klik op de link die daarin staat om je emailadres te bevestigen.

Bijna alles op GitHub is gepubliceerd met een open source licentie. Dat betekent dat de auteur toestemming geeft aan iedereen om zijn of haar code te kopiëren, aan te passen, opnieuw te publiceren en er eventueel geld aan te verdienen. Als je de auteur van iets bent – of dat nou een boek, tekening of software is – heb je het auteursrecht, en mag jij beslissen hoe andere mensen dat werk gebruiken.

Van heel veel software die je gebruikt kan je de broncode niet bekijken; deze is closed source. Bijvoorbeeld iOS op je telefoon, Windows op je computer, en de meeste apps of programma's die je gebruikt. Bedrijven en auteurs willen vaak hun producten niet open source maken, omdat ze bang zijn dat de concurrentie met hun idee aan de haal gaat of zelfs kloont.

Dat bijna alles op GitHub open source is, heeft wel het effect dat mensen sneller geneigd zijn met elkaar samen te werken. Ze bieden hun wijzigingen vaak aan bij de oorspronkelijke auteur om het project verbeteren. Dat werkt met een pull request (PR). Eerst maak je een kopie van het project (een fork), dan maak je wijzigingen in de kopie (met een commit), en daarna open je een pull request om de wijzigingen voor te stellen aan de oorspronkelijke auteur.

De website die je nu aan het lezen bent is ook open source, en je kan wijzigingen voorstellen met een PR. Dus zie je (spel)fouten of heb je een sugesstie? Maak een PR of issue aan.

GitHub zelf is overigens niet open source. Er is sinds een aantal jaar wel een goed open source alternatief: Gitlab. Het programma Git is van een andere auteur (Linus Torvalds, ook de ontwikkelaar van Linux), en wel altijd open source geweest.
